medusa.recon.mpipe.mpipe#

Module with a wrapper around a Mediapipe face mesh model 1 that can be used in Medusa.

1

Kartynnik, Y., Ablavatski, A., Grishchenko, I., & Grundmann, M. (2019). Real-time facial surface geometry from monocular video on mobile GPUs. arXiv preprint arXiv:1907.06724

get_faces()#
__call__(image)#

Performs reconstruction of the face as a list of landmarks (vertices).

Parameters

image (np.ndarray) – A 3D (w x h x 3) numpy array representing a RGB image

Returns

out – A dictionary with two keys: "v", the reconstructed vertices (468 in total) and "mat", a 4x4 Numpy array representing the local-to-world matrix

Return type

dict

Notes

This implementation returns 468 vertices instead of the original 478, because the last 10 vertices (representing the irises) are not present in the canonical model.

Examples

To reconstruct an example, simply call the Mediapipe object:

>>> from medusa.data import get_example_frame
>>> model = Mediapipe()
>>> img = get_example_frame()
>>> out = model(img)  # reconstruct!
>>> out['v'].shape    # vertices
(468, 3)
>>> out['mat'].shape  # local-to-world matrix
(4, 4)
close()#

Closes context manager.

Ideally, after you’re doing with reconstructing each frame of the video, you call this method to close the manually opened context (but shouldn’t matter much if you only instantiate a single model).
